# t-agent 产品建设工作空间入口

## 1. 工作空间定位

本目录不是泛知识库，也不是单次演讲材料目录。它是 t-agent / 企业级 Data Agent 的长期产品建设工作空间。

旧目录 `idealization/5月/` 的角色是历史输入源：保留 5 月期间围绕企业 Data Agent、DB-GPT、ChatExcel、ChatBI、ChatReport、知识工作台、Agent Harness、ToolSearch 等主题形成的材料。后续主工作区从根目录展开。

## 2. 当前 Product Committee 决策

- Decision: `narrow`
- Product shape: 先把 t-agent 收敛为“企业数据分析 Agent 平台能力 + 垂直入口模块”的可验证路线，而不是立即开一个大而全的工程项目。
- Current SSOT: `02-roadmap/t-agent-roadmap.md`
- Agent SSOT entry: `agent.md`
- V2 scope: Dataset Learning、Knowledge Base、现有 ChatBI 集成、ChatExcel 单文件/先单表的分析报告迭代。
- Candidate workflow: 销售经营分析可以作为 golden workflow / eval pack，但不再定义 V2。
- Non-goals: V2 不先做完整 ChatReport、Dashboard Insight、Skill Hub、Asset Hub、Feishu 发布、企业多租户、自动行动型 Agent。
- Main risk: 材料很多但未形成当前版本边界，容易再次滑向“概念完整、交付不收敛”。

## 3. 历史材料主线

历史材料可以被归成六条主线：

| 主线 | 核心结论 | 主要来源 |
|---|---|---|
| 产品定位 | Data Agent 不是聊天框，而是企业数据分析能力操作系统 | `gather/基本共识.md`, `transsion-enterprise-data-agent-sharing-draft.md` |
| 平台能力 | DB-GPT / t-agent 应承担 Runtime、应用门户、公共能力中心、适配框架 | `gather/v2/产品规划v1.md`, `enterprise-buildups.md` |
| 可信问数 | 企业级问数必须建设数据集学习、语义指标、SQL 安全、运行反馈 | `enterprise-buildups.md`, `datafun-speaker-submission-data-agent.md` |
| 报告链路 | 经营分析报告是高价值 wedge，必须保留 SQL、图表、指标和知识引用 | `transsion-enterprise-data-agent-sharing-draft.md`, `gather/v2/v2.0需求收敛与版本边界.md` |
| 资产沉淀 | 数据集、字段语义、问题模板、Skill、报告素材、运行记录要成为平台资产 | `gather/基本共识.md`, `gather/能力清单.md` |
| Harness / 知识更新 | Skill 不是 prompt；能力需要触发、资源、工具、评测、版本和候选发现机制 | `_workbench/sessions/2026-06-09-Anthropic-Skills-官方仓库与-harness-思想.md`, `_workbench/sessions/2026-06-13-ToolSearch与知识更新机制审计.md` |

## 4. 当前推荐建设方式

1. 根目录作为长期 t-agent workspace。
2. `idealization/5月/` 保留为历史输入和归档源，不再作为 Codex 主项目入口。
3. 新材料先进入 `01-product/`、`02-roadmap/`、`03-architecture/`、`06-iteration/`。
4. 需要引用旧材料时，在 `04-sources/source-register.md` 登记来源、版本、类型和关联，不复制长文。
5. 需要正式改变产品方向时，在 `05-decisions/` 新增 ADR。
