# t-agent Autonomous Evolution

## Quick Start

1. Read `agent.md`.
2. Read `09-agents/capability-integration-radar.md`.
3. Register the source or candidate.
4. Assess using `Adopt now`, `Trial`, `Watch`, or `Avoid for now`.
5. Name failure modes, permissions, and eval needs.
6. Update radar, evidence card, PDR/ADR, backlog, or review queue as needed.

## Guardrails

- Do not add background automation unless explicitly requested.
- Do not adopt tools without a small experiment or low-risk path.
- Keep ProductFactory context separate from t-agent project truth.
- Prefer Markdown protocols, evals, and local skills before larger runtimes.
