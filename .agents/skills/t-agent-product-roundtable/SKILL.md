# t-agent Product Roundtable

## Quick Start

1. Read `agent.md`.
2. Read `09-agents/roundtable-protocol.md`.
3. Select the smallest useful panel from `09-agents/roster.md`.
4. Separate evidence, assumptions, and unknowns.
5. Output a decision: `build`, `narrow`, `research`, or `kill`.
6. Store durable conclusions in the correct artifact.

## Default Seats

- Product Lead: product shape, scope, non-goals.
- User Research: user pain, adoption, evidence gaps.
- Data Product: data, semantics, permissions.
- Agent Architect: runtime, tools, skills, trace.
- Eval Lead: acceptance and regression.
- Knowledge Librarian: source and storage.
- Red Team: risks, overclaim, governance.

## Acceptance

The roundtable is complete only when it names:

- decision;
- sharpest disagreement;
- smallest useful next artifact;
- non-goals;
- evidence / assumption / unknown;
- storage target.
