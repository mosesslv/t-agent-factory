# Obsidian V1 视图说明

## 目标

让 `00-知识工作台入口.md` 从导航页升级为工作台状态页。

## 推荐路径

V1 优先使用 Dataview 查询块，因为它可以直接在 Markdown 文件中维护，迁移成本低。

`00-知识工作台入口.md` 的首屏应优先展示当前工作台总览和审阅队列，而不是只保留静态链接列表：

```markdown
![[_index/generated-workspace-dashboard#Snapshot]]
![[_index/generated-workspace-dashboard#Decision Focus]]
![[_index/generated-review-queue#Top Tasks]]
```

工作台总览由 `tools/kb workspace-dashboard` 生成，适合人阅读；浏览器只读视图由 `_index/generated-workspace-ui.html` 承接；人工做审阅动作前先读 `[[_index/generated-decision-guide]]`；检索和 RAG 边界由 `[[_index/generated-rag-scope]]` 承接；完整命令列表由 `[[_index/generated-review-queue#Decision Commands]]` 承接；Agent 和插件读取 `_index/generated-workspace-dashboard.json`、`_index/generated-decision-guide.json`、`_index/generated-rag-scope.json` 与 `_index/generated-review-queue.json`。

如已启用 Obsidian Bases，可进一步把以下视图做成 Bases：

- 最近研讨 session
- 待沉淀候选
- 已 promoted 正式知识
- deprecated / superseded 内容

## Dataview 查询

最近研讨：

```dataview
TABLE status, topic, stage, updated
FROM "_workbench/sessions"
WHERE type = "discussion-session"
SORT updated DESC
```

待沉淀候选：

```dataview
TABLE status, topic, promotion_status, target_topic_path, updated
FROM "_workbench/distillations"
WHERE type = "distillation-candidate" AND promotion_status != "promoted"
SORT updated DESC
```

正式知识：

```dataview
TABLE status, topic, confidence, updated
FROM "主题库"
WHERE type = "knowledge-card"
SORT updated DESC
```

决策记录：

```dataview
TABLE status, topic, updated
FROM "主题库"
WHERE type = "decision-record"
SORT updated DESC
```

过时内容：

```dataview
TABLE type, topic, updated
FROM ""
WHERE status = "deprecated"
SORT updated DESC
```
