# 知识入库规则

## 入库边界

正式知识库是 `主题库/`。只有经过提炼、审阅、确认的内容才能进入。

工作台是 `_workbench/`。工作台内容默认只是过程材料，不代表正式结论。

## 可以进入正式知识库的内容

- 反复讨论后形成的稳定判断。
- 明确概念定义。
- 可复用框架或方法论。
- 决策记录。
- 共识记录。
- 标注置信度和来源的假设。
- 典型案例。
- 外部证据和来源。

## 只应留在工作台的内容

- 原始语音或文字输入。
- 发散讨论过程。
- 红蓝攻防原文。
- 临时想法。
- 未验证假设。
- 被推翻但仍有参考价值的中间过程。
- Agent 提示词、角色过程、输出草稿。
- 一次性上下文。

## Promote 前检查

写入 `主题库/` 前必须检查：

- 是否有清晰主题。
- 是否有来源 session。
- 是否有状态和置信度。
- 是否与已有知识重复或冲突。
- 是否需要标记旧知识为 `deprecated`。
- 是否已经得到用户确认。

## 受控 Promote 流程

V2 起，正式入库使用两段式命令：

```bash
tools/kb promote --dry-run <distillation-file>
tools/kb check-distillation <distillation-file>
tools/kb preview-promote <distillation-file>
tools/kb review-distillation <distillation-file> approve --note "人工确认理由"
tools/kb promote --confirm <distillation-file>
```

`approve-distillation` 仍可作为兼容命令使用，但推荐使用 `review-distillation`，因为它会追加审阅记录。

约束：

- `--dry-run` 不写文件。
- `check-distillation` 检查来源、目标目录、稳定判断、概念定义、决策候选、待验证问题和人工确认说明。
- `preview-promote` 预览将创建或追加的文件、生成 ID、内容计数和目标文件冲突。
- `review-distillation ... approve` 代表人工确认已经发生。
- `review-distillation ... approve` 默认要求 `check-distillation` 通过。
- `--confirm` 只接受 `promotion_status: approved-for-promotion` 的候选。
- `--confirm` 会写入正式主题库文件，并把候选标记为 `promoted`。

## 审阅状态流

来源资料审阅：

```bash
tools/kb check-source <source-note>
tools/kb prepare-source <source-note>
tools/kb draft-source-summary <source-note>
tools/kb apply-source-draft <source-note> --confirm
tools/kb preview-source-review <source-note>
tools/kb decision-guide --kind source-note --limit 3
tools/kb review-source <source-note> reviewed --note "可作为来源"
tools/kb review-source <source-note> needs-evidence --note "缺少证据"
tools/kb review-source <source-note> rejected --note "不采用"
```

`prepare-source` 会把 `check-source` 未通过项写入 `## 补全任务`。`draft-source-summary` 只写入候选草稿，不替代人工审阅。`apply-source-draft` 默认只预览，`--confirm` 才会把候选草稿复制到正式 `## 摘要` 和 `## 初步判断`；复制后可用 `preview-source-review` 只读查看审阅材料和建议决定命令，仍需要 `review-source ... reviewed` 才算人工审阅通过。`review-source ... reviewed` 默认要求 `check-source` 通过；来源资料必须有关联 session、摘要和初步判断。

沉淀候选审阅：

```bash
tools/kb check-distillation <distillation-file>
tools/kb preview-distillation-review <distillation-file>
tools/kb decision-guide --kind distillation-candidate --limit 3
tools/kb review-distillation <distillation-file> approve --note "同意进入 promote 前置状态"
tools/kb review-distillation <distillation-file> needs-revision --note "需要继续打磨"
tools/kb review-distillation <distillation-file> reject --note "不进入正式知识库"
```

`decision-guide` 只读展示每个待审任务的证据摘要、可选动作、适用条件、风险和 note 模板。它不代表人工确认。`preview-distillation-review` 只读展示沉淀内容、来源 session、目标文件计划、冲突风险、检查项和建议决定命令。审阅命令只更新状态和追加 `## 审阅记录`，不会自动写入 `主题库/`。

## 正式知识 frontmatter

建议正式知识卡使用：

```yaml
---
id: kc-YYYYMMDD-001
type: knowledge-card
topic: topic-id
status: draft
confidence: medium
created: YYYY-MM-DD
updated: YYYY-MM-DD
source_sessions:
  - ws-YYYYMMDD-001
tags:
  - tag
---
```

决策记录建议使用：

```yaml
---
id: adr-YYYYMMDD-001
type: decision-record
topic: topic-id
status: accepted
created: YYYY-MM-DD
updated: YYYY-MM-DD
source_sessions:
  - ws-YYYYMMDD-001
supersedes: []
---
```
