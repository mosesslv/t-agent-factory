# 工作流说明

## 总原则

Knowledge Workbench 的职责是让人和 Agent 共同完成：

```text
输入碎片 → 调取已有知识 → 多专家研讨 → 沉淀候选 → 人确认 → 正式知识
```

Codex、Claude Code、ChatGPT、Cursor 或本地脚本都只是客户端。系统记录以 Markdown、frontmatter、索引和 Git 为准。

## 客户端启动

任何 Agent 或脚本进入当前项目时，先执行或读取：

```bash
tools/kb agent-brief
tools/kb agent-brief --json
tools/kb tool-catalog --json
tools/kb decision-guide --limit 3
```

`agent-brief` 是只读接手上下文，包含工作区根目录、版本状态、首读入口、当前最高优先级审阅任务、promote 审计摘要、安全规则和推荐命令。它不执行 review、approve 或 promote。

`tool-catalog` 是工具边界目录，声明命令、资源、权限级别、写入范围、风险和人工确认要求。未来 MCP server 或其他 Agent 客户端应先读取该目录，再决定哪些工具可以暴露或自动调用。

`decision-guide` 是人工审阅前的只读决策手册，列出证据摘要、可选动作、适用条件、风险和 note 模板。它不代表人工确认。

如果不能执行 CLI，则先读：

- `00-知识工作台入口.md`
- `_index/generated-agent-brief.md`
- `_index/generated-tool-catalog.md`
- `_index/generated-decision-guide.md`
- `_index/generated-review-queue.md`
- `_index/generated-review-packet.md`
- `_meta/知识入库规则.md`
- `_meta/版本管理规则.md`

## 议题启动

当用户提出战略研讨或知识沉淀需求时：

1. 明确议题名称。
2. 判断相关已有知识位置。
3. 从 `主题库/`、`gather/`、`plan/` 或 `_index/` 读取必要上下文。
4. 在 `_workbench/sessions/` 创建 session，或继续已有 session。
5. 如果本轮需要红蓝攻防、多专家质询或跨轮收敛，基于已有 session 创建 `_workbench/debates/` 辩论文件。
6. 明确本轮输出目标：探索、攻防、收敛、沉淀候选或 promote。

## 研讨过程

多专家讨论建议使用以下角色：

- 产品经理：定义用户、场景、需求、优先级。
- 战略专家：判断趋势、组织定位、长期价值。
- 架构专家：判断系统边界、数据结构、技术路线。
- 产品专家：判断 MVP、交互闭环、用户体验。
- 红队专家：质疑假设、指出空泛和不可落地。
- 知识编辑专家：提炼知识卡、决策记录、待验证假设。
- 研发交付专家：处理版本、回退、排期、验收。

每轮专家输出应包含：

- Claim：核心观点
- Evidence：依据
- Risk：反对意见或风险
- Question：需要进一步验证的问题
- Recommendation：是否建议进入沉淀候选

## 沉淀流程

1. 原始输入进入 `_workbench/inbox/` 或 session。
2. 来源资料通过 `tools/kb link-source <source-note> <session-id-or-path>` 关联到具体研讨 session。
3. 研讨过程进入 `_workbench/sessions/` 和 `_workbench/debates/`。
4. 辩论成熟后，先运行 `preview-debate-review` 审阅，再用 `draft-distillation-from-debate` 生成 `_workbench/distillations/` 草稿。
5. 阶段性结论进入 `_workbench/distillations/`。
6. 人确认后，才允许写入 `主题库/`。
7. promote 后，正式知识卡必须记录来源 session。

## 禁止事项

- 不把原始对话直接写入正式知识库。
- 不把提示词模板写入正式知识正文。
- 不把未验证假设伪装成共识。
- 不覆盖旧结论来制造“最新正确版本”。
- 不在没有用户确认的情况下 promote。

## 推荐状态

| 状态 | 含义 |
|---|---|
| inbox | 原始输入或待处理碎片 |
| draft | 已整理但未确认 |
| reviewed | 人已审阅但未必正式入库 |
| promoted | 已进入正式知识库 |
| deprecated | 已过时或被更新结论替代 |
