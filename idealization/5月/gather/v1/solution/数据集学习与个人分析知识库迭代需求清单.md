## 1. 迭代目标

本次迭代目标是在 DB-GPT 页面内建设一套可被 ChatExcel 和 ChatBI 共同消费的公共能力：

1. 数据集选择与数据集学习。
2. 字段语义识别、字段注释编辑、快捷问题初始化。
3. 个人分析知识上传、正文拉取、解析、检索和问数上下文注入。
4. ChatExcel 和 ChatBI 双入口体验。
5. 运行留痕和验收数据记录。

当前 MVP 的核心不是分别增强两个应用，而是验证：

> 同一套数据集学习资产和个人分析知识资产，能否同时支撑 ChatBI 的问指标/问数场景，以及 ChatExcel 的问表/明细分析/报告总结场景。

## 2. 场景边界

### 2.1 双入口定位

| 入口 | 主要用户场景 | 数据消费方式 | 结果形态 |
|---|---|---|---|
| ChatBI | 问指标、问数、经营口径解释 | 消费 StarRocks 中相对标准化的业务宽表，重点使用指标、维度、口径和过滤条件 | SQL、指标结果、趋势解释、推荐追问 |
| ChatExcel | 问表、看明细、做分析报告总结 | 消费 Excel/CSV 或 StarRocks 中选定表，重点使用字段、明细、聚合结果和分析框架 | 表格、图表、明细解释、分析总结 |

### 2.2 数据源范围

本期支持两类数据源：

1. **文件数据源**：Excel / CSV。
2. **数据库数据源**：StarRocks。

StarRocks 本期需要开发数据选择页面和能力：

- 支持选择库。
- 支持选择表。
- 支持选择相对标准化的业务宽表进行学习和分析。
- MVP 优先支持 PSI 周粒度宽表、PSI 月粒度宽表。
- PSI 周/月宽表在 ChatBI 中已有对应数据基础，ChatExcel 和 ChatBI 都可以消费这两张表，但应用场景不同。

### 2.3 个人分析知识范围

本期个人分析知识库支持：

- 用户在 ChatExcel 或 DB-GPT 内 ChatBI 对话框中上传知识。
- 支持 Word 文档。
- 支持飞书云文档正文拉取。
- 知识内容可包含指标口径、字段解释补充、分析思路、领域知识、分析框架、业务规则、历史报告等。
- 默认仅本人可见。
- 当前会话允许共享使用。
- MVP 暂不考虑组织/租户隔离。

## 4. P0 需求明细

### REQ-001 数据源选择入口

用户故事：

作为业务用户，我希望在 ChatExcel 或 DB-GPT 内 ChatBI 中可以选择要分析的数据来源，包括上传文件和选择数据库表，这样我可以围绕同一批业务数据进行问数或问表分析。

功能要求：

- 提供数据源选择入口。
- 支持 Excel/CSV 上传入口。
- 支持 StarRocks 数据库入口。
- 用户进入 ChatExcel 或 ChatBI 时，都可以选择已注册或可访问的数据集。

验收标准：

- 用户可以在页面上看到文件数据源和 StarRocks 数据源入口。
- 用户可以完成一个 Excel/CSV 数据集注册。
- 用户可以完成一个 StarRocks 库表选择。

### REQ-002 StarRocks 库表选择

用户故事：

作为业务用户，我希望可以从 StarRocks 中选择库和表，优先选择 PSI 周/月粒度宽表，这样我可以在 ChatBI 中问指标，也可以在 ChatExcel 中围绕同一张表做明细分析和总结。

功能要求：

- 对接 StarRocks 连接。
- 支持选择数据库。
- 支持选择表。
- MVP 优先暴露 PSI 周粒度宽表和 PSI 月粒度宽表。
- 选表后生成 Dataset 资产。
- Dataset 资产可被 ChatExcel 和 DB-GPT 内 ChatBI 共用。

验收标准：

- 用户可以选择 PSI 周粒度宽表。
- 用户可以选择 PSI 月粒度宽表。
- 同一张表可以进入 ChatBI 问数流程。
- 同一张表可以进入 ChatExcel 问表/分析总结流程。

待确认：

- StarRocks 连接配置、库名、表名、测试账号由研发或数据负责人补充。
- PSI 周/月宽表字段清单由产品经理或数据负责人补充。

### REQ-003 Excel/CSV 数据集注册

用户故事：

作为业务用户，我希望上传 Excel/CSV 后系统自动注册为数据集，这样我可以继续进行字段学习、字段注释编辑和问表分析。

功能要求：

- 支持 Excel/CSV 上传。
- 支持生成 Dataset。
- 支持识别 sheet、字段名、字段类型、样例值。
- 支持进入数据集学习任务。

验收标准：

- 上传文件后可以看到数据集详情。
- 系统能展示字段列表和样例值。
- 数据集可以被 ChatExcel 消费。

### REQ-004 标准宽表数据集学习

用户故事：

作为业务用户，我希望系统能自动学习 PSI 周/月宽表的字段含义、维度、指标和时间粒度，这样我可以用自然语言进行问数或问表分析。

功能要求：

- 支持 StarRocks 宽表字段学习。
- 支持 Excel/CSV 字段学习。
- 输出字段中文名、字段解释、语义类型、维度/度量判断、时间字段、是否可问。
- 支持学习任务状态：未学习、学习中、学习成功、学习失败。

验收标准：

- 对 PSI 周/月宽表可以生成字段学习结果。
- 字段学习结果可被 ChatBI 和 ChatExcel 读取。
- 学习失败时有明确错误状态。

### REQ-005 字段语义识别与字段画像

用户故事：

作为业务用户，我希望系统能识别字段的业务含义、字段类型、样例值和可用范围，这样 AI 在回答问题时能减少字段误用。

功能要求：

- 识别字段类型。
- 抽取样例值、空值率、Top 值。
- 判断字段是维度、指标、时间、枚举、文本或标识字段。
- 生成字段解释建议。

验收标准：

- 字段列表中可展示语义类型和字段解释。
- ChatBI 问指标时能读取指标/维度上下文。
- ChatExcel 问表时能读取字段解释和明细上下文。

### REQ-006 字段注释人工编辑

用户故事：

作为业务用户，我希望可以手动调整字段注释和同义词，这样系统后续问数和问表时能按我的业务理解解释字段。

功能要求：

- 支持编辑字段注释。
- 支持编辑字段同义词。
- 支持设置字段是否可问。
- 保存后进入数据集学习资产。

验收标准：

- 用户编辑字段注释后，后续问数上下文使用新注释。
- 用户设置不可问字段后，问数链路不主动使用该字段。

### REQ-007 快捷问题生成与编辑

用户故事：

作为业务用户，我希望系统基于数据集初始化一些可以问的问题，这样我能快速开始分析。

功能要求：

- 基于字段学习结果生成快捷问题。
- 支持按 ChatBI 问指标场景生成指标类问题。
- 支持按 ChatExcel 问表/总结场景生成明细分析类问题。
- 支持用户编辑、删除、排序。

验收标准：

- PSI 周/月宽表至少生成 5 条快捷问题。
- ChatBI 展示偏指标口径的问题。
- ChatExcel 展示偏明细分析和总结的问题。

### REQ-008 ChatExcel 消费数据集学习资产

用户故事：

作为 ChatExcel 用户，我希望可以选择已学习的数据集，围绕表字段进行问表、明细分析和报告总结。

功能要求：

- ChatExcel 支持选择已注册数据集。
- 支持消费字段注释、字段同义词、字段画像、快捷问题。
- 支持消费个人分析知识片段。
- 支持生成 SQL 或分析逻辑并返回表格、图表、总结。

验收标准：

- 用户可以在 ChatExcel 中选择 PSI 周/月宽表。
- 用户可以问明细、做汇总分析、生成分析总结。
- 回答能结合字段学习结果和个人分析知识。

### REQ-009 DB-GPT 内 ChatBI 应用入口

用户故事：

作为 ChatBI 用户，我希望在 DB-GPT 页面内进入 ChatBI 应用，选择同一批数据集进行问指标和问数分析。

功能要求：

- 在 DB-GPT 页面内提供 ChatBI 应用入口。
- 支持选择数据集。
- 支持展示快捷问题。
- 支持对话式问数。
- 支持上传个人分析知识。

验收标准：

- 用户可以从 DB-GPT 页面进入 ChatBI。
- 用户可以选择 PSI 周/月宽表并发起问数。
- ChatBI 问数结果可以在 DB-GPT 页面内展示。

### REQ-010 ChatBI Adapter

用户故事：

作为平台研发，我希望通过 Adapter 对接 ChatBI 服务，而不是把 ChatBI 核心代码迁入 DB-GPT，这样可以保持 ChatBI 服务独立，同时让它消费 DB-GPT 的公共数据集上下文。

功能要求：

- 封装 ChatBI 调用。
- 注入 `dataset_context`。
- 注入 `personal_knowledge_context`。
- 适配 ChatBI 返回结果。
- 异常时返回明确错误信息。

验收标准：

- DB-GPT 可以通过 Adapter 调用 ChatBI。
- ChatBI 可以接收字段上下文和个人知识上下文。
- ChatBI 返回结果能进入 DB-GPT 统一展示和运行记录。

### REQ-011 ChatBI 消费数据集上下文

用户故事：

作为 ChatBI 用户，我希望 ChatBI 能理解我选择的数据集字段、指标和口径，这样问数时能减少字段匹配错误。

功能要求：

- ChatBI 请求中携带 `dataset_uid`。
- 携带字段目录、字段注释、同义词、指标/维度角色。
- 携带快捷问题或推荐问题。
- 支持 PSI 周/月宽表上下文。

验收标准：

- ChatBI 能基于选择的数据集回答指标类问题。
- ChatBI 回答中能体现字段注释或指标口径。

### REQ-012 对话框上传个人分析知识

用户故事：

作为业务用户，我希望在 ChatExcel 或 ChatBI 对话框里点击“上传知识”，上传自己的分析文档，这样系统可以按我的指标口径、字段解释和分析框架回答问题。

功能要求：

- 在 ChatExcel 对话框提供上传知识入口。
- 在 DB-GPT 内 ChatBI 对话框提供上传知识入口。
- 支持上传 Word。
- 支持输入飞书云文档链接并拉取正文。
- 支持展示上传状态。
- 默认仅本人可见。
- 当前会话允许共享使用。

验收标准：

- 用户可以上传 Word 文档。
- 用户可以输入飞书云文档链接并完成正文拉取。
- 上传后的知识可以在当前会话中被问数链路使用。

### REQ-013 Word 文档解析与知识切片

用户故事：

作为业务用户，我希望系统能解析我上传的 Word 文档，识别其中的指标口径、字段解释和分析思路，这样我不需要手工复制粘贴知识。

功能要求：

- 解析 Word 正文。
- 进行分段、切片。
- 抽取标题、段落、表格文本。
- 生成可检索知识片段。
- 记录来源文档。

验收标准：

- Word 文档上传后能生成知识片段。
- 知识片段保留来源文档名称。
- 问数时能检索到相关片段。

### REQ-014 飞书云文档正文拉取

用户故事：

作为业务用户，我希望可以上传飞书云文档链接，让系统直接拉取正文内容，这样团队已有业务口径文档可以进入问数上下文。

功能要求：

- 支持输入飞书云文档链接。
- 使用已有飞书开放平台权限拉取正文。
- 拉取失败时展示原因。
- 拉取成功后进入知识切片和索引流程。

验收标准：

- 有权限的飞书文档可以拉取正文。
- 拉取正文后可生成知识片段。
- 无权限或链接错误时有明确提示。

待确认：

- 飞书应用配置、权限范围、测试文档链接由产品经理或平台负责人补充。

### REQ-015 个人知识检索

用户故事：

作为业务用户，我希望系统在回答问题时自动查找我上传过的相关知识，这样回答可以符合我的业务口径和分析习惯。

功能要求：

- 支持按用户、会话、数据集检索知识。
- 默认仅检索本人知识。
- 当前会话内可共享使用。
- 返回 TopN 相关知识片段。
- 支持上下文压缩，避免注入过长。

验收标准：

- 问题命中相关指标口径时，能检索到对应知识片段。
- 回答能体现知识片段中的口径或分析框架。

### REQ-016 问数上下文注入

用户故事：

作为业务用户，我希望系统在回答时同时参考数据集学习结果和个人知识，这样既不会问错字段，也能按业务口径解释结果。

功能要求：

- 构造统一问数上下文。
- 上下文包含 `dataset_context`。
- 上下文包含 `personal_knowledge_context`。
- ChatExcel 和 ChatBI 都能消费该上下文。

验收标准：

- ChatExcel 问表时能结合字段注释和个人知识。
- ChatBI 问指标时能结合指标口径和个人知识。

### REQ-017 运行留痕 QueryRun

用户故事：

作为产品和研发，我希望每次问数都记录使用了哪些字段、哪些知识、生成了什么 SQL 和结果，这样后续可以复盘准确率和问题原因。

功能要求：

- 记录用户问题。
- 记录入口：ChatExcel 或 ChatBI。
- 记录数据集。
- 记录命中字段。
- 记录命中知识片段。
- 记录 SQL 或分析逻辑。
- 记录结果摘要。
- 记录反馈。

验收标准：

- 每次问数都有 QueryRun 记录。
- 可以查看字段命中和知识命中。
- 可以区分 ChatExcel 和 ChatBI 来源。

## 9. 宣讲会前必须对齐

1. 本期不是只做 ChatExcel，而是 ChatExcel + DB-GPT 内 ChatBI 双入口 MVP。
2. 本期不是做 ChatBI 既有模板化分析，而是做个人分析知识上传、解析、检索和上下文注入。
3. 本期数据源不只包含 Excel/CSV，还包含 StarRocks 选库选表。
4. PSI 周/月粒度宽表是 MVP 优先支持的数据集。
5. ChatBI 主要验证问指标/问数，ChatExcel 主要验证问表/明细分析/报告总结。
6. 个人知识默认仅本人可见，当前会话允许共享，MVP 不做组织/租户隔离。
7. 飞书云文档本期需要拉取正文，不只是登记链接。
8. 验收样例由产品经理补充占位内容。

## 10. 验收口径

MVP 完成需要满足：

1. 用户可以在 DB-GPT 页面中从 ChatExcel 和 ChatBI 两个入口进入。
2. 用户可以上传 Excel/CSV 并完成数据集学习。
3. 用户可以选择 StarRocks 中的 PSI 周/月宽表并完成数据集学习。
4. 用户可以编辑字段注释和同义词。
5. 系统可以生成快捷问题。
6. 用户可以在对话框上传 Word 文档。
7. 用户可以输入飞书云文档链接并拉取正文。
8. 系统可以检索个人分析知识片段。
9. ChatExcel 可以结合数据集学习和个人知识进行问表、明细分析和总结。
10. ChatBI 可以结合数据集学习和个人知识进行问指标、问数和解释。
11. 每次问数都能记录数据集、字段命中、知识命中、SQL 或分析逻辑、结果和反馈。
