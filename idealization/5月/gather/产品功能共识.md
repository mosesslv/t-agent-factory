 **Data Agent 产品能力 Landscape v0.1** 。

拆成两个 L0：

| L0             | 名称                                                  | 定义                                                                                                                                            |
| -------------- | ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| **L0-A** | **通用平台能力 / Common Platform Capabilities** | 所有数据应用共用的底座能力，包括 Data Center、Knowledge Center、Dataset Learning、Semantic Layer、Agent Runtime、权限、安全、评估、资产沉淀等。 |
| **L0-B** | **垂直数据应用 / Vertical Data Applications**   | 面向具体用户任务的应用入口，例如 ChatExcel、ChatData/ChatDB、ChatBI、ChatDashboard、ChatReport 等。                                             |

这版的核心原则是：

> **平台能力不归属于某一个应用；ChatExcel、ChatBI、ChatReport 等只是平台能力的不同消费端。**

这与当前附件中的阶段方案是一致的：附件里已经明确建议在 DB-GPT / Data Agent 平台侧建设数据集学习、数据集资产、字段语义、问题模板、个人分析知识库、运行留痕、评估反馈等公共能力，而不是把这些能力做成 ChatExcel 或 ChatBI 的私有能力。

## 概念区分


## 2.1 Data Center 不是传统数仓，而是 Data Agent 的“数据资产控制面”

传统数据中台/数仓关注的是数据接入、加工、建模、治理、服务化。Data Agent 里的 Data Center 需要继承这些能力，但更进一步要服务于 AI 问数、分析、报告和 Agent 调度。

所以 Data Center 在这里不是“又造一个数仓”，而是：

> **让 Data Agent 知道企业有哪些数据、数据在哪里、字段是什么意思、质量如何、谁能用、怎么用。**

DAMA 的数据管理体系也强调数据资产、元数据、质量、安全、集成、治理、DW/BI 等都是数据管理的核心知识域。

你已有的数据平台架构材料中也出现了数据接入、数据开发、数据治理、数据质量、数据标准、数据安全、资产目录、血缘、DataOps、数据服务市场等模块，这些都可以映射到 Data Agent 的平台能力层。

## 2.2 Dataset Learning Center 和 Data Learning Center 要拆开

你当前提到的 **data learning center** 建议拆成两个概念：

| 概念      | 建议命名                           | 说明                                                                                  |
| --------- | ---------------------------------- | ------------------------------------------------------------------------------------- |
| AI 学数据 | **Dataset Learning Center**  | 让系统学习表、字段、样例值、质量、可问范围、指标维度、推荐问题。                      |
| 人学分析  | **Analysis Learning Center** | 给用户学习分析方法、案例、框架、优秀报告、官方技能。可放在 Asset Hub / Skill Hub 里。 |

## 2.3 Knowledge、Semantic、Skill、Asset 四者不能混

这四个概念建议这样区分：

| 名称                | 解决什么问题                 | 典型对象                                                       |
| ------------------- | ---------------------------- | -------------------------------------------------------------- |
| **Knowledge** | 业务知识是什么               | 指标口径文档、分析框架、业务规则、行业知识、历史报告           |
| **Semantic**  | 数据应该如何被机器正确使用   | 指标、维度、字段别名、同义词、默认过滤条件、聚合规则           |
| **Skill**     | 一类分析任务怎么做           | 趋势分析技能、同比环比分析技能、贡献度分析技能、经营报告技能   |
| **Asset**     | 分析过程产生了什么可复用成果 | 数据集、图表、洞察卡片、报告素材、SQL、Prompt、案例、Dashboard |
