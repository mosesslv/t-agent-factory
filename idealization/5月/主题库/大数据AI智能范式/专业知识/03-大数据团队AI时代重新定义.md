# 大数据团队 AI 时代重新定义

## 新定义

大数据团队不再只是“取数、建表、做报表”的交付团队，而是企业智能分析能力提供方。

```text
大数据团队
= AI-ready 数据资产提供者
+ 业务语义和指标口径治理者
+ Agent 工具与行动边界设计者
+ 分析 Skill 和报告素材运营者
+ Run Trace / Eval / AgentOps 负责人
```

## 职责变化

| 过去职责 | 未来职责 |
|---|---|
| 做数仓表 | 提供可问、可解释、可追溯的数据产品 |
| 做 BI 报表 | 提供图表、洞察、报告素材和智能解读能力 |
| 接取数需求 | 提供受控问数 Skill 和数据服务 API |
| 管数据权限 | 管数据访问 + Agent 工具调用 + 行动审计 |
| 维护口径文档 | 维护可执行语义层和指标版本 |
| 人工排查错误 | 建立 QueryRun、EvalCase、失败回放和持续优化 |

## 新能力要求

- 数据产品化：Dataset Card、Owner、质量、刷新、使用范围。
- 语义工程：指标口径、维度、同义词、默认过滤、时间语义。
- Agent 工程：Planner、Tool Calling、Handoff、Structured Output。
- 治理工程：SQL Guard、MCP Gateway、行列权限、敏感字段。
- 评测工程：黄金问题集、回归测试、语义正确性评估。
- 资产运营：Skill Hub、Asset Hub、案例库、报告模板。

## 关键组织判断

业务分析师不会被大数据团队替代，大数据团队也不会被业务自助工具替代。新的分工是：

```text
业务侧：提出问题、解释业务、做判断、推动行动
数据侧：提供可信数据、语义、工具、治理、评测和资产沉淀
Agent：承担检索、查询、归因、草拟、编排和执行辅助
```
