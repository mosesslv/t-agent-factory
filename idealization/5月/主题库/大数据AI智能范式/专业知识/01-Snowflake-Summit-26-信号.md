# Snowflake Summit 26 信号

## 核心判断

Snowflake Summit 26 的主线不是“Snowflake 增加了一批 AI 功能”，而是 Snowflake 正在把自己重新定义为企业 AI Agent 的数据、语义、治理、安全和执行底座。

## 五个信号

1. **AI on governed data**  
   模型进入数据平台，敏感数据不离开受治理环境。

2. **Agent 成为数据平台的新用户**  
   CoWork 服务知识工作者，CoCo 服务 builder，二者都以 Agent 方式消费数据、上下文和工具。

3. **语义层成为 Agent 可信上下文层**  
   Horizon Context 把业务定义、语义视图、BI 定义、治理上下文统一起来，降低 Agent 误解指标的风险。

4. **治理从数据访问扩展到 Agent 行动**  
   Natoma MCP Gateway 意味着治理对象扩展到工具调用、身份授权、策略执行和审计。

5. **开放湖仓成为 AI 控制面基础**  
   Iceberg、Polaris、Zero-Copy、外部引擎访问管理指向跨系统、单一受治理数据副本。

## 对本地 Data Agent 建设的启示

不要先做孤立聊天机器人，而要建设：

```text
Data Agent Harness
+ Governed Semantic Runtime
+ Action Governance
+ Run Trace & Eval
```

否则 Agent 越强，数据误解、越权访问、错误行动和不可追溯风险越大。
