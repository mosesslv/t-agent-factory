## Obsidian 查询建议

如果启用 Dataview，可以直接使用下面的状态视图。

### 最近研讨

```dataview
TABLE status, topic, stage, updated
FROM "_workbench/sessions"
WHERE type = "discussion-session"
SORT updated DESC
```

### 最近专家辩论

```dataview
TABLE status, topic, stage, updated
FROM "_workbench/debates"
WHERE type = "debate-session"
SORT updated DESC
```

### 待沉淀候选

```dataview
TABLE status, topic, promotion_status, target_topic_path, updated
FROM "_workbench/distillations"
WHERE type = "distillation-candidate" AND promotion_status != "promoted"
SORT updated DESC
```

### 来源资料

```dataview
TABLE status, topic, review_status, source_type, updated
FROM "_workbench/inbox"
WHERE type = "source-note"
SORT updated DESC
```

### 正式知识卡

```dataview
TABLE status, confidence, updated
FROM "主题库"
WHERE type = "knowledge-card"
SORT updated DESC
```

### 决策记录

```dataview
TABLE status, topic, updated
FROM "主题库"
WHERE type = "decision-record"
SORT updated DESC
```

### 过时内容

```dataview
TABLE type, topic, updated
FROM ""
WHERE status = "deprecated"
SORT updated DESC
```

## CLI

V1 提供轻量本地 CLI：

```bash
tools/kb status
tools/kb status --json
tools/kb version-status
tools/kb agent-brief
tools/kb agent-brief --json
tools/kb tool-catalog
tools/kb tool-catalog --json
tools/kb workspace-dashboard
tools/kb workspace-dashboard --json
tools/kb workspace-ui > _index/generated-workspace-ui.html
tools/kb decision-guide
tools/kb decision-guide --json
tools/kb rag-scope
tools/kb rag-scope --json
tools/kb queue
tools/kb queue --json
tools/kb review-queue
tools/kb review-packet
tools/kb validate
tools/kb search "Agent"
tools/kb refresh-index
tools/kb new-session "议题名称"
tools/kb new-debate "议题名称红蓝攻防" --session ws-20260608-001 --target-topic-path 主题库/主题目录/
tools/kb check-debate _workbench/debates/2026-06-09-议题名称红蓝攻防.md
tools/kb preview-debate-review _workbench/debates/2026-06-09-议题名称红蓝攻防.md
tools/kb draft-distillation-from-debate _workbench/debates/2026-06-09-议题名称红蓝攻防.md
tools/kb new-topic "主题名称" --topic topic-id
tools/kb new-source "来源标题" --url "https://example.com" --topic topic-id
tools/kb link-source _workbench/inbox/2026-06-08-来源标题.md ws-20260608-002
tools/kb check-source _workbench/inbox/2026-06-08-来源标题.md
tools/kb prepare-source _workbench/inbox/2026-06-08-来源标题.md
tools/kb draft-source-summary _workbench/inbox/2026-06-08-来源标题.md
tools/kb apply-source-draft _workbench/inbox/2026-06-08-来源标题.md --confirm
tools/kb preview-source-review _workbench/inbox/2026-06-08-来源标题.md
tools/kb review-source _workbench/inbox/2026-06-08-来源标题.md reviewed --note "摘要和来源可信"
tools/kb promote --dry-run _workbench/distillations/2026-06-08-人和Agent共同编织知识网络-沉淀候选.md
tools/kb check-distillation _workbench/distillations/2026-06-08-人和Agent共同编织知识网络-沉淀候选.md
tools/kb preview-promote _workbench/distillations/2026-06-08-人和Agent共同编织知识网络-沉淀候选.md
tools/kb promote-audit _workbench/distillations/2026-06-08-人和Agent共同编织知识网络-沉淀候选.md
tools/kb preview-distillation-review _workbench/distillations/2026-06-08-人和Agent共同编织知识网络-沉淀候选.md
tools/kb review-distillation _workbench/distillations/2026-06-08-人和Agent共同编织知识网络-沉淀候选.md approve --note "人工同意进入正式入库前置状态"
tools/kb promote --confirm _workbench/distillations/2026-06-08-人和Agent共同编织知识网络-沉淀候选.md
```

CLI 默认只做状态、队列、校验、dry-run 和 preview。`version-status` 只读展示 Git 版本状态；`queue` 会给出 `next_action`、`preview_command` 和 `decision_commands`；`review-queue` 会按优先级输出当前最该处理的人工审阅任务；`apply-source-draft` 默认只预览，只有 `--confirm` 才写入来源正式段落；`preview-source-review` 和 `preview-distillation-review` 是只读人工审阅页；`review-source ... reviewed` 默认要求 `check-source` 通过；`review-distillation ... approve` 默认要求 `check-distillation` 通过；之后仍要显式执行 `promote --confirm` 才会写入正式知识库。

`review-packet` 会把当前最高优先级人工审阅任务的上下文、检查项和建议命令汇总成只读审阅包，不会执行任何确认动作。

`promote-audit` 会在正式入库前展示门禁、计划写入文件、Git 状态、建议提交边界和回退方式，不会执行 approve 或 promote。
