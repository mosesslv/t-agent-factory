## 建议入库内容

### 稳定判断

1. 当前项目要建设的不是传统笔记库，而是人和 Agent 共同编织的知识网络。
2. 人在系统中主要承担议题发起、价值判断、确认入库和阅读复盘职责。
3. Agent 在系统中主要承担已有知识调取、外部资料研究、多专家辩论、结构化提炼和索引维护职责。
4. 工作台与正式知识库必须隔离；工作台内容默认不是正式知识。
5. V0 应优先使用 Markdown、Obsidian、Git 和模板，而不是直接建设复杂 Agent runtime。

### 概念定义

1. 知识库层：给人和 Agent 长期使用的稳定认知网络，承载知识卡、决策记录、来源证据和主题索引。
2. 工作台层：人和 Agent 共同讨论、试错、辩论、提炼的运行空间，承载 session、debate、hypothesis 和 distillation。
3. Promote：经过人工确认后，将沉淀候选写入正式知识库的动作。

### 可复用框架

```text
输入碎片
→ 工作台 session
→ 多专家研讨
→ 红队质疑
→ 沉淀候选
→ 人工确认
→ 正式知识卡 / 决策记录
→ Git 可追踪回退
```

### 决策记录候选

1. V0 不迁移现有旧知识，只新增工作台结构和索引。
2. V0 不引入 LangGraph、Mastra、Vercel AI SDK 或 Web App。
3. Promote 必须人工确认，不允许自动将工作台内容写入正式知识库。
4. 旧知识被替代时优先标记 deprecated，不直接删除。

## 不建议入库内容

- 原始聊天全文。
- 子 Agent 的中间过程。
- 未验证的工具热度判断。
- 对具体框架的强结论，除非经过实际试用或更系统评估。
